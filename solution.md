# 方案说明（Solution Overview）

## 1. 总体架构
本项目旨在通过图神经网络与大语言模型（LLM）结合，实现高价值客户线索的自动发现与分析。整体流程分为数据加载、拓扑规则选择、候选节点检索、批量评分、规则发现与叙述生成六大步骤。

## 2. 主要流程

### 2.1 数据加载
- 通过 `load_artifacts` 从 Excel 文件读取节点与边，构建 `networkx.DiGraph` 有向图。
- 同时加载拓扑规则（topology.md），并封装为 `GraphArtifacts`。

### 2.2 拓扑规则选择
- 用户可在界面选择或编辑拓扑规则，影响后续检索与评分。
- 选中的规则通过 `with_topology` 注入到 `GraphArtifacts`，用于后续 LLM 检索与评分。

### 2.3 候选节点检索
- `retrieve_candidates` 利用 cross-encoder 语义检索模型，对所有节点和边与拓扑规则进行相关性打分。
- 选出 top-K 作为候选客户节点。

### 2.4 批量评分
- `score_batch` 将候选节点的上下文信息与拓扑规则一同输入 LLM，批量获得每个节点的得分与理由。
- 得分高于阈值的节点进入最终目标客户列表。

### 2.5 规则发现
- `discover_rules` 针对高分节点，自动归纳新的图结构规则，辅助后续优化。
- 新规则可自动追加到 topology.md。

### 2.6 节点叙述生成（核心分析）
- 选中目标节点后，`analyze_node` 首先通过 `_collect` 收集该节点的多跳邻域子图。
- `_classify` 批量调用 LLM 对所有边进行分类（如 CONTROL, FAMILY, EVENT 等），并标注优先级与时效性。
- `_threads` 根据分类结果自动聚合出多条结构化“线索线程”，如控制链、家族链、风险事件等。
- `_serialize` 将所有线程按优先级拼接为上下文摘要。
- 最后将摘要与目标节点、业务目标一同输入 LLM，生成面向业务的中文叙述（insight）与推荐行动（recommended_action）。

## 3. 主要技术要点
- **图结构建模**：采用 networkx.DiGraph，节点与边属性灵活。
- **LLM 边分类**：完全由 LLM 分类边类型与优先级，摒弃传统关键词匹配。
- **多线程聚合**：自动聚合多条业务相关的“线索线程”，提升叙述丰富度。
- **端到端自动化**：从数据加载、检索、评分、规则发现到叙述生成全流程自动化。

## 4. 典型调用链
1. 用户上传 Excel，加载图与规则。
2. 选择拓扑规则，检索候选节点。
3. LLM 批量评分，筛选高分目标。
4. 选中目标节点，LLM 分类边并聚合线程，生成业务叙述与行动建议。

## 5. 适用场景
- 财富管理、家族办公室、企业客户关系梳理、风险合规等需要复杂关系洞察的场景。
